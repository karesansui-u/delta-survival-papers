# Experiment 39: Prospective Contradiction-Dominance Test

This is a small prospective validation of the structural-persistence prediction:

> A short context with task-relevant structural contradiction should perform worse
> than a much longer context containing only task-irrelevant filler.

The experiment uses the same three-variable addition task and target sets as
Exp.36, but with a minimal 2×2 prospective design and a separately registered
primary contrast. Exp.36 is prior evidence; Exp.39 is meant to be a focused
replication of the key contradiction-dominance prediction rather than a new task
family.

## Design

2 contradiction conditions × 2 context lengths × 30 trials:

| Factor | Levels |
|---|---|
| Contradiction | `zero` filler-only, `structural` self-contradictory task-variable constraints |
| Context length | 32K, 256K |
| Primary contrast | `32K structural` vs `256K zero` |

## Primary Prediction

`accuracy(32K structural) < accuracy(256K zero)`.

The prediction is directional. A large margin, for example at least 20 percentage
points, is treated as strong support. If `32K structural` is not worse than
`256K zero`, the prospective prediction fails.

## Files

- `exp39_prospective_contradiction_dominance.py`: runner, dry-run, and summarizer
- `exp39_preregistration.md`: frozen design, predictions, and falsification rule
- `exp39_{model}_trials.jsonl`: generated by the runner

## Pilot Note

An initial held-out expression variant, `2*a - b + c + d`, was attempted before
the primary contrast. It failed the sanity criterion because `32K zero` accuracy
was `0/30` for `gpt-4.1-nano`, before any structural-condition trials were run.
Those partial data were moved to
`exp39_failed_expression_pilot_gpt-4_1-nano_trials.jsonl` and are not treated as
the Exp.39 primary test.

A second pilot with new three-variable target values also failed the intended
sanity role because `32K zero` had target-specific arithmetic errors. Those data
were moved to `exp39_failed_new_targets_pilot_gpt-4_1-nano_trials.jsonl`. The
final Exp.39 runner therefore uses the Exp.36 target sets exactly, isolating the
prospective contrast to context length and contradiction condition.

## Usage

Dry-run, no API calls:

```bash
python3 analysis/exp39/exp39_prospective_contradiction_dominance.py dry-run
```

Run paid API calls explicitly:

```bash
python3 analysis/exp39/exp39_prospective_contradiction_dominance.py run --execute
```

Summarize completed trials:

```bash
python3 analysis/exp39/exp39_prospective_contradiction_dominance.py summarize
```
