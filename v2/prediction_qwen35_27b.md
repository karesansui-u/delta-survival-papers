1. 理論的根拠

構造持続の最小形式（Paper 1）は、制約の蓄積を構造維持可能な状態集合の縮小として記述する。
Paper 3 の補助命題は、制約の質によって効き方が異なると予測する。

  補助命題: 構造的矛盾は事実的衝突よりも広く有効経路を削り、
  論理一貫性維持率をより強く悪化させる。

この補助命題から、代謝介入（矛盾解消）の効果は、構造的矛盾を直接扱う
指標ほど大きく現れるはずである。


2. gemma4:31b での既知パターン（rejudge 済み）

|---------------------|------|------|----------|

| fact_recall (adj)   | 47%  | 40%  | +7pt     |

この順序は補助命題と整合する。
構造的矛盾（contradiction）への介入効果が最大であり、
事実的衝突（fact）への効果が最小。

qwen ファミリーでも contra/rule が高く fact が低いパターンが

3. qwen3.5:27b に対する予測

  予測 P1: ON > OFF（全体）

  予測 P2: カテゴリ別効果サイズの順序は contra_det > rule > fact
  これは補助命題の直接的帰結であり、モデル非依存で成立するはずである。

  予測 P3: contra_det で ON-OFF 差が最大になる
  P2 の系として、最も強いシグナルは contradiction_detection に現れる。

3.2 境界条件の予測

  予測 P4: fact_recall は旧 expected で全条件低い（< 20%）
  context pollution の構造的限界（モデル非依存）により、
  fact_recall は代謝の有無にかかわらず低水準にとどまる。
  contra-adjusted expected で再判定した場合も ON ≈ OFF と予測する。
  F3 矛盾注入は L1 に直接残るため、代謝の有無に非依存。

  予測 P5: NC 条件は contra_det で本来 0% に近いはず
  矛盾が注入されないため検出対象が存在せず、
  ベンチマークの contradiction_detection パラドックス
  （「矛盾はありません」が正解だが FAIL 判定される）が発生する。
  gemma4:31b でも NC contra_det = 0% だった。
  ただし keyword judge が「矛盾はありません」等の否定文から
  「矛盾」を拾うため、見かけ上の高スコアが出る可能性がある。
  この場合、理論の棄却ではなく judge の偽陽性として扱う。

3.3 予測しないもの

  - 絶対値の具体的な数字（モデル能力・プロンプト感受性に依存）
  - rule_application の ON-OFF 差の正確な大きさ
  - qwen 固有の failure mode（thinking mode 問題等）

以下を満たせば「理論から導かれた予測が独立なモデルで支持された」と言える。

  (f) OFF > ON が全体で成立する → 理論の前提を再検討
  (g) fact の ON-OFF 差 > contra_det の ON-OFF 差 → 補助命題の棄却

6.1 外れの原因分析

P1-P3 が外れた原因は三つ。

第一に、qwen3.5:27b は contra_det タスク自体が苦手。
gemma4:31b は矛盾を指摘する応答を生成するが、qwen3.5:27b は
「情報がないので確認できません」系の応答を返す。
ON/OFF 以前に、矛盾検出の基礎能力が低い（床効果）。

第二に、代謝パイプラインの abstraction drift。
代謝が知識を抽象化する過程で具体的な属性値を落としており、
rule_application で ON が OFF を下回る逆転が生じた。
これは代謝実装の問題であり、理論の前提の問題ではない。

第三に、代謝 LLM の質がボトルネックであった可能性。
後続の smoke 実験（2026-04-11）で、同一の対話 LLM（qwen3.5:9b）に
対して代謝 LLM を Sonnet に切り替えたところ、矛盾検出が 2/2 成功
した。qwen-only 代謝では L3 に矛盾記録が正しく入らず、下流の対話
LLM が矛盾を報告できなかった。したがって、第一の原因（対話モデル
の基礎能力不足）は部分的に見直される。contra_det の失敗は対話モデル
単体の能力閾値ではなく、代謝パイプラインに入る LLM の質に依存する
可能性がある。

6.2 結論

qwen3.5:27b は主仮説に対して negative / inconclusive な結果。
ただし理論の全否定ではなく、以下の境界条件の発見として扱う。

- 理論の予測が成立するには、対話モデルの基礎能力だけでなく、
  代謝 LLM の質にも閾値がある。gemma4:31b では両方を超えていたが、
  qwen3.5:27b を代謝 LLM にも使う構成では代謝側が不足していた
  可能性がある。後続実験で代謝 LLM を Sonnet に切り替えたところ、
  同一の対話 LLM（qwen3.5:9b）でも contra が動いた。
- 現在の代謝パイプラインは矛盾検出には有効だが、
  抽象化時に具体値を保持しないため rule に逆効果が生じうる。
- P4（fact 低水準）と P5（NC contra ≈ 0%）は支持された。
  これらはモデル能力に依存しない構造的予測であり、
  モデル横断で頑健に成立している。

論文への記載方針:
- 「事前予測が完全に的中した」とは書けない
- 「contra_det の代謝効果は、対話モデルと代謝 LLM の両方の能力に依存する」と書ける
- 「代謝 LLM を高能力モデルに切り替えると contra が回復する」は
  Paper 5（混成構成）の三層分離を直接支持する経験的証拠になる
- 「abstraction drift は代謝パイプラインの改善課題として残る」と書ける
- P4・P5 の的中は「モデル非依存の構造的予測」として記載可能

7.6 真の原因の特定

当初、§6 の外れの原因を (1) 代謝 LLM 品質、(2) Router フィルタ、(3) 応答生成の三層として分析した。しかし、再実験の準備過程で真の原因が判明した。

**真の原因: num_ctx ハードコード**

`src/adapters/ollama_llm.py` が num_ctx=4096 でハードコードされており、実際のプロンプトは 7000+ トークンに達していたため、冒頭の「現在の世界モデル」セクション（L3 検索結果 = 矛盾記録）が silently 切り捨てられていた。

検証:
- 実験時の prompt_eval_count は全ターンで 4096 に張り付いていた
- ローカルで同じプロンプトを num_ctx=16384 で投げると、qwen3.5:27b は矛盾を完璧に検出する
- retrieval、prompt 構築、代謝パイプライン、案 C（contradiction bypass）は全て正常に動いていた
- 案 C は smoke test（短いプロンプト）では効いたが、フル実験では num_ctx がボトルネックだった

修正: `OLLAMA_NUM_CTX` 環境変数を導入、デフォルト 16384

この発見により、§6.1 で挙げた三つの原因のうち、第 2（Router フィルタ）と第 3（応答生成）は実際の主因ではなかったことになる。第 1（代謝 LLM 品質）は smoke test で確認済みなので独立の要因として残るが、フル実験での主要ボトルネックは num_ctx だった。

7.7 この結果の意味

(d) が外れた（contra > fact > rule となり fact と rule が逆転）のは、rule が元々 80% と高く ON-OFF の改善余地が小さかったこと（天井効果）で説明できる。補助命題の核心である「構造的矛盾への介入効果が最大」は成立しており、rule と fact の相対順序は天井効果に依存する二次的な問題である。

P5r（fact < 20%）が大幅に外れたのは、num_ctx 修正の副次効果である。prompt 切り捨てが解消されたことで fact 情報も正しく prompt に入るようになり、事実再現率が 0% から 76% へと劇的に改善した。これは構造的な限界と思われていた fact_recall の低水準が、実装の問題（num_ctx 切り捨て）に起因していた可能性を示唆する。
